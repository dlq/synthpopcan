# SynthPopCan Branding Assets

This directory contains the project logo, icon, and favicon assets. The PNG files are transparent unless noted otherwise.

## Documentation Logo

Use `logo/synthpopcan-logo-512.png` for most README and documentation pages. Use the 1024 px version only when a larger source image is useful, and the 256 px version for compact layouts.

- `logo/synthpopcan-logo-master.png`: original accepted transparent logo asset.
- `logo/synthpopcan-logo-source.png`: square padded transparent source used to create resized exports.
- `logo/synthpopcan-logo-1024.png`: high-resolution documentation logo.
- `logo/synthpopcan-logo-512.png`: default documentation logo.
- `logo/synthpopcan-logo-256.png`: compact documentation logo.

## Icon And Favicon

Use `icon/favicon.ico` for browser favicon support. Use the matching PNG size when a specific platform asks for one.

- `icon/synthpopcan-icon-master.png`: transparent icon source derived from the documentation logo.
- `icon/synthpopcan-icon-source.png`: square padded transparent source used to create resized exports.
- `icon/synthpopcan-icon-512.png`
- `icon/synthpopcan-icon-256.png`
- `icon/synthpopcan-icon-128.png`
- `icon/synthpopcan-icon-64.png`
- `icon/synthpopcan-icon-32.png`
- `icon/synthpopcan-icon-16.png`
- `icon/favicon.ico`: multi-size favicon bundle.

## Source File Notes

Keep the `*-master.png` files as preserved upstream assets. Use the `*-source.png` files when regenerating size-specific PNGs so future exports keep the same square canvas and transparent padding.

The `icon/` assets intentionally come from the same can/logo family as the documentation logo, rather than a separate redrawn mark, so the favicon stays visually consistent with the main branding.
